## 1. 背景と目的

### 1.1 背景
依頼者は、瞬読（速読）と記憶法を鍛えるための Web サイトを実装したいと考えている。すでに以下2つの HTML を自作している。

- `瞬読トレーニングアプリ_4.html`：フラッシュ／単語／円形／ランダム配置の4トレーニング＋右脳沈黙モード＋履歴グラフ
- `Memory-trainer-advance_number.html`：数字記憶トレーニング＋スコア／正答率／称号／折れ線・棒グラフ

これらを統合・改良し、解説コンテンツを加えた一体型サイトにしたい。

参考にしたコンテンツ:
- 瞬読：『瞬読ドリル』（SBクリエイティブ）特典動画ページ。「観察力トレーニング」「イメージ力トレーニング」が柱。
- 記憶法：ドミニク・オブライエン（世界記憶力選手権8回優勝）の記憶術書のレビュー記事。記憶を「才能」ではなく「技術」として扱い、イメージ化・場所への配置・回収という考え方が中心。

### 1.2 目的
1. 瞬読と記憶法を「学べて」「鍛えられる」一体型サイトを提供する。
2. 各トレーニングに **意義（なぜ効くのか）** と **効果的なやり方・手順** を併記し、ただ機能を使うだけでなく原理を理解できるようにする。
3. 既存トレーニングの良さ（履歴・グラフ・称号など）を引き継ぎつつ、デザインと内容を拡充する。

## 3. スコープ

### 3.1 対象範囲（In Scope）
- 単一 HTML ファイルによる統合サイト（瞬読／記憶法の2本柱＋ホーム）
- 各トレーニングの解説（意義・やり方・手順）／ほどほどに簡潔な文章トーン
- 瞬読トレーニング6種、記憶法トレーニング5種
- **瞬読・記憶法の実力チェックテスト**（レベル判定・称号付与の起点）
- **瞬読サンプルテキストの GitHub 読み込み＋ドロップダウン選択**
- 学習履歴の記録・可視化（グラフ・表・CSV）・称号

### 3.2 対象外（Out of Scope）
- サーバーサイド処理・ユーザー認証・クラウド同期（データは端末内に保存）
- マルチユーザー対応（**完全な個人運用・単独利用**を前提とする）
- 課金・有料コンテンツ（**完全無料運用**）
- ネイティブアプリ化
- 参考書籍・記事本文の転載（解説は独自の言葉で記述する）

```
│   ├─ サイトの目的・使い方
│   ├─ 瞬読とは / 記憶法とは（概要）
│   └─ 各セクションへの導線（CTA）
├─ ［瞬読］… カード一覧（メニュー）→ タップで詳細画面
│   ├─ 意義とやり方
│   ├─ 共通設定（フォント倍率・右脳沈黙モード・サンプルテキスト選択）
│   ├─ フラッシュ / 単語 / 円形 / ランダム配置 / 観察力 / イメージ力
│   ├─ 実力チェックテスト
│   └─ 記録（履歴・グラフ）
└─ ［記憶法］… カード一覧（メニュー）→ タップで詳細画面
    ├─ 意義（記憶は技術である）
    ├─ 記憶術の解説（場所法 / ジャーニー法 / ストーリー法）※解説のみ
    ├─ 数字記憶 / 単語記憶 / カード配置記憶 / ドミニク
    ├─ 実力チェックテスト（種目選択）

各詳細画面の共通レイアウト：
  ［← 一覧へ］＋タイトル → 意義・やり方の解説 → トレーニング本体（A案）
```

> テーマはライトのみ。詳細画面はカードから開くスライドイン演出で表示し、「← 一覧へ」でメニューに戻る。

## 5. 瞬読セクション 機能要件

### 5.1 解説コンテンツ
- **意義**：黙読（音声化）を抑え、文字を「形・かたまり」として捉えることで読書速度を上げる。右脳的な全体把握・観察力・イメージ化が鍵であること。
- **効果的なやり方・手順**：1日数分でよいこと、毎日の継続が重要なこと、慣れたら速度・文字数を上げる段階的進め方、「正確に読む」より「印象で掴む」意識づけ など。
- 各トレーニングカードの冒頭にも、そのトレーニング固有の意義・やり方を短く併記する。

### 5.2 トレーニング一覧

| # | 名称 | 概要 | 素材 | 由来 |
|---|---|---|---|---|
| 1 | フラッシュトレーニング | テキストを文字数／文／段落単位で高速表示 | 読込テキスト | 既存改良 |
| 2 | 単語トレーニング | 単語を高速で次々表示し瞬間認識を鍛える。**横書き／縦書き切替**、**1文字ずつ順番をシャッフル** | 内蔵単語リスト | 既存改良 |
| 3 | 円形トレーニング | 文字を円状に配置し一瞬表示、全体視野を鍛える。**文字順はシャッフル** | 内蔵単語リスト | 既存改良 |
| 4 | ランダム配置トレーニング | 文字を画面内にランダム配置し一瞬表示 | 内蔵単語リスト | 既存改良 |
| 5 | 観察力トレーニング | 数字のグリッド（2×2〜4×4）を一瞬表示し再現 | 自動生成 | 新規（『瞬読ドリル』参照） |
| 6 | イメージ力トレーニング | 複数の語を瞬間表示→イメージ化→想起 | 内蔵語 | 新規（『瞬読ドリル』参照） |

> **内蔵単語リスト**：日常で使う語を中心に、**1語あたり4〜10文字**のものだけを収録（プロトタイプ時点で約270語）。フラッシュのみ「共通設定」で読み込んだテキストを題材とする。

### 5.3 共通設定・機能
- **フォント倍率スライダー**（0.8〜2.5）：文字を大きくして「形の印象」を強める（黙読抑制）。
- **右脳沈黙モード**：刺激のあと黒画面に切り替え、数秒間「言語化せず印象だけ味わう」時間をつくる。秒数設定可。
- **サンプルテキスト選択（GitHub連携）**：GitHub 上で管理するテキスト群をドロップダウンから選んで読み込む。これを各トレーニングの題材に使う。
- テキスト入力は **GitHub 選択／テキストエリア貼り付け／.txt ファイル読込** の3経路に対応する（既存はファイルのみ）。
- GitHub やファイルが使えない場合に備え、最低限の内蔵サンプルもフォールバックとして持つ。

### 5.4 実力チェックテスト（瞬読）
- 一定条件（決まった文字数・表示時間など）で出題し、瞬間認識・再現の正確さからレベルを判定する。
- 結果を履歴に記録し、達成基準に応じて称号を付与する。
- 詳細仕様は「9.5 実力チェックテスト仕様（案）」を参照。

### 5.5 改良ポイント（既存からの変更）
- ファイル必須だった点を解消し、GitHub選択・貼り付け・未準備でも開始できるようにする。
- **カード起点UIに刷新**：各トレーニングはメニューカードから個別の詳細画面として開く。
- 単語・円形・ランダムを**内蔵単語リスト（4〜10字）**由来に変更し、単語・円形は**文字順をシャッフル**。単語に**縦書き／横書き切替**を追加。
- 入力ガード・停止挙動・タイマー表示を整理。

## 6. 記憶法セクション 機能要件

### 6.1 解説コンテンツ
- **意義**：記憶力は生まれつきの才能ではなく訓練可能な「技術」である。覚えにくい情報をイメージに変換し、既知の場所に配置し、順番にたどって回収する、という設計の考え方。
- **共通原理**：イメージ化／場所への配置／回収経路／奇妙さ（印象に残る変なイメージ）／連想／集中・観察。
- 各トレーニングカードの冒頭に、その手法の意義と使いどころ・手順を併記する。

### 6.2 トレーニング／解説の構成

記憶法セクションは「**解説のみの項目**」と「**トレーニングを持つ項目**」に分かれる。

**解説のみ（記憶術の解説）**

| 名称 | 内容 |
|---|---|
| 場所法／記憶の宮殿 | 説明＋「想像力／創造力／脳への変化」の記述（トレーニングは持たない） |
| ジャーニー法 | 同上 |
| ストーリー法・連想 | 同上 |

**トレーニングを持つ項目**

| # | 名称 | 概要 | 状態 |
|---|---|---|---|
| 1 | 数字記憶 | 桁数・表示秒数を設定し、数字列を記憶して再現・採点 | 既存改良 |
| 2 | 単語記憶 | 内蔵単語リストから出題し、記憶して想起・採点 | 新規 |
| 3 | カード配置記憶 | トランプの全カード配置を覚え、伏せた後にペアの位置を当てる | 新規 |
| 4 | ドミニクシステム | 数字→文字→人物・行動・物への変換を学び・ドリルで練習 | 新規 |

> 当初案にあった場所法・ジャーニー法・ストーリー法の「対話型トレーニング」は、依頼により**解説のみ**に変更（想像力・創造力・脳への変化を文章で記述）。代わりに**単語記憶**と**カード配置記憶**を実装する。

### 6.3 各トレーニングの主な仕様

#### 6.3.1 数字記憶（改良）
- 入力：桁数、表示秒数、区切り表示（なし／2桁／3桁ごと）の有無。
- 流れ：表示→カウントダウン→非表示→入力→採点（位置一致数・正答率・スコア）。
- 記録：正答率・桁数・秒数・スコアを履歴へ。

#### 6.3.2 単語記憶
- 入力：単語数、記憶時間（秒）。
- 流れ：内蔵単語リストから出題 → 記憶タイム → 非表示 → 想起入力（改行／読点区切り）→ 一致数・正答率・スコアを採点（順不同）。
- 用途：ストーリー法・場所法を実際に試す場としても使える。

#### 6.3.3 カード配置記憶（旧・神経衰弱）
- 絵柄：トランプ（♠♥♦♣の A〜K）。同じ**ランク**でペア成立。
- 入力：ペア数（4／6／8／10／13）、記憶時間（秒）。
- 流れ：**全カードの表を一定秒数表示 → 一斉に伏せる → 記憶を頼りに2枚ずつめくってペアを当てる**。
- 採点：手数（理想＝ペア数）から効率・スコアを算出して履歴へ。

#### 6.3.4 ドミニクシステム
- 学習：ドミニク式の数字→文字変換表（0=O,1=A,2=B,3=C,4=D,5=E,6=S,7=G,8=H,9=N）を提示。
- 個人テーブル：00〜99 に対応する「人物・行動・物」をユーザーが登録（localStorage 保存、**初期サンプルは入れず空から段階的に入力**）。
- ドリル：登録済みの番号を出題し、対応する人物を想起できるか練習・採点。
- 解説：なぜ数字を人物・行動・物に変換すると覚えやすいのかを併記。

### 6.4 実力チェックテスト（記憶法）
- **種目選択式**：数字記憶（20桁・30秒）／単語記憶（15語・40秒）／カード配置記憶（8ペア・手数）から選んで実施。
- 結果を履歴に記録し、成績に応じて称号を付与する。
- 詳細仕様は「9.5 実力チェックテスト仕様（案）」を参照。

### 6.5 共通機能
- 履歴・グラフ（手法別／日別・月別）・スコア表・CSV 出力・履歴リセット。
- 称号（既存ロジックを基に、記憶法全体の平均スコア等で付与）。

## 7. 共通・非機能要件

### 7.1 デザイン
- **明るく落ち着いた学習サイト風**。可読性重視の配色・余白・タイポグラフィ（明朝見出し＋ゴシック本文）。
- 瞬読＝深緑、記憶法＝テラコッタでアクセント色を分け、現在地が直感的に分かるようにする。
- **カード起点ナビ**：各セクションはメニューカードの一覧から、個別の詳細画面を開いて使う。
- **動的アニメーション**：カードの時間差フェードイン、ホバーでの浮き上がり、詳細画面のスライドイン、トランプカードのポップ／フリップなどのマイクロインタラクションを付与。
- **ライトのみ**で統一（ダークテーマは提供しない）。

### 7.2 技術・制約
- 単一 HTML ファイル（HTML + CSS + JavaScript を内包）。
- 外部依存はグラフ描画ライブラリ（Chart.js／CDN）程度に限定。
- データ保存は **localStorage**（端末内）。サーバー不要。
- レスポンシブ（PC／タブレット／スマホ）。
- **完全な個人運用・単独利用・完全無料**。ホスティングは GitHub Pages 等の静的ホスティングを想定。
- **瞬読サンプルテキストは GitHub リポジトリで管理**し、`raw.githubusercontent.com` から fetch して読み込む。テキスト一覧はマニフェスト（例：`texts/index.json`）で管理し、ドロップダウンに反映する。今後 Claude Code でテキスト／スキルを追加していく前提。

### 7.3 データ・プライバシー
- 履歴・設定・ドミニク個人テーブルは端末内 localStorage のみに保存。
- CSV 出力でバックアップ可能。履歴リセットで全削除可能（確認ダイアログ付き）。

### 7.4 アクセシビリティ／使い勝手
- 文字サイズ・コントラストに配慮。
- 開始前にサンプルや初期値を用意し、すぐ試せる状態にする。
- 操作ボタンの状態（開始中／停止中）を明示。

## 8. データ設計（概要）

localStorage キー（案）:

| キー | 内容 |
|---|---|
| `shun_history` | 瞬読の履歴（日時・種類・時間・速度／結果 等） |
| `mem_history` | 記憶法の履歴（日時・手法・正答率・スコア 等）。実力テスト結果も手法名を分けて同居 |
| `dominic_table` | ドミニクシステムの個人テーブル（00〜99） |

> プロトタイプでは瞬読・記憶法それぞれ1本の履歴キーに集約し、実力テストも手法名（例：`実力テスト(数字)`）で区別して記録。設定値（フォント倍率・沈黙秒数等）は現状フォーム値で保持。将来 `shun_settings` 等への永続化は任意。
> 旧キー（`trainingHistory` / `digit_history`）からの移行は **不要**（新規キーで開始）。

### 8.1 GitHub サンプルテキスト（外部データ）
- リポジトリ内に `texts/` を用意し、各テキストを `.txt` で保存。
- `texts/index.json` にメタ情報を列挙（例）:

```json
[
```

### 9.5 実力チェックテスト仕様（プロトタイプ実装）

> 称号付与の起点となる、条件を固定した「腕試し」コンテンツ。

**瞬読 実力チェック（実装）**
- 数字列を一瞬（約0.6秒）表示。3桁から始まり、正解するごとに1桁ずつ増加。
- 失敗時点の到達桁数でレベル・称号を判定（例：9桁以上＝一瞬の達人）。

**記憶法 実力チェック（実装・種目選択）**
- 数字記憶：20桁を30秒で記憶→再現。位置一致数で称号。
- 単語記憶：15語を40秒で記憶→想起。一致数で称号。
- カード配置記憶：8ペアの配置を覚え、最少手数でクリア。手数で称号。

**共通**
- 結果（到達レベル／正答数・手数・日時）を履歴に記録し、推移グラフ・称号に反映。
